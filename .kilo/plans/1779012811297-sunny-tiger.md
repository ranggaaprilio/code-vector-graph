# Plan: Merge Jina + Nomic Embedding Model Support

## Problem

The `main` branch has **nomic-embed-code** as the only embedding model (3584 dims, no task prefixes, float16). The `feature/jina-code-embeding` branch replaced it with **jina-code-embeddings-1.5b** (1536 dims, task prefixes, bfloat16). We need to merge both so users can choose which model to use.

## Key Differences Between Branches

| Aspect | Main (Nomic) | Feature (Jina) |
|--------|-------------|----------------|
| Model | `nomic-ai/nomic-embed-code` | `jinaai/jina-code-embeddings-1.5b` |
| Dimensions | 3584 | 1536 |
| Tokenizer | `nomic-ai/nomic-embed-code` | `jinaai/jina-code-embeddings-1.5b` |
| Chunk prefixes | None (no `search_document:`) | `JINA_TASK_PREFIXES` (code2code/nl2code) |
| Query prefixes | None | `JINA_TASK_PREFIXES["nl2code"]["query"]` |
| Torch dtype | float16 | bfloat16 (float32 for MPS/ROCm) |
| Chunker | No `_split_long_line` | Has `_split_long_line` for token-boundary splitting |
| Tests | 3584/768 dims | 1536 dims |

## Implementation Plan

### 1. `src/config.py` ‚Äî Define multi-model configuration

Replace the single-model config with a `MODEL_CONFIGS` dict containing both models, and add a `DEFAULT_MODEL_ID` constant:

```python
MODEL_CONFIGS = {
    "nomic": {
        "model_name": "nomic-ai/nomic-embed-code",
        "dimensions": 3584,
        "tokenizer_name": "nomic-ai/nomic-embed-code",
        "dtype": "float16",
        "prefixes": None,  # No task prefixes for nomic
    },
    "jina": {
        "model_name": "jinaai/jina-code-embeddings-1.5b",
        "dimensions": 1536,
        "tokenizer_name": "jinaai/jina-code-embeddings-1.5b",
        "dtype": "bfloat16",
        "prefixes": {
            "code2code": {
                "query": "Find an equivalent code snippet given the following code snippet:\n",
                "passage": "Candidate code snippet:\n",
            },
            "nl2code": {
                "query": "Find the most relevant code snippet given the following query:\n",
                "passage": "Candidate code snippet:\n",
            },
        },
    },
}

DEFAULT_MODEL_ID = "nomic"

# Backward-compatible aliases that resolve from MODEL_CONFIGS
DEFAULT_MODEL = MODEL_CONFIGS[DEFAULT_MODEL_ID]["model_name"]
EMBEDDING_DIMENSIONS = MODEL_CONFIGS[DEFAULT_MODEL_ID]["dimensions"]
TOKENIZER_NAME = MODEL_CONFIGS[DEFAULT_MODEL_ID]["tokenizer_name"]

EMBEDDING_PROVIDERS = {
    "huggingface": {
        "model": MODEL_CONFIGS[DEFAULT_MODEL_ID]["model_name"],
        "dimensions": MODEL_CONFIGS[DEFAULT_MODEL_ID]["dimensions"],
    },
}

def get_model_config(model_id: str) -> dict:
    """Resolve model config by ID (e.g., 'nomic' or 'jina')."""
    if model_id not in MODEL_CONFIGS:
        raise ValueError(f"Unknown model_id '{model_id}'. Available: {list(MODEL_CONFIGS.keys())}")
    return MODEL_CONFIGS[model_id]
```

### 2. `src/cli.py` ‚Äî Add `--model` CLI argument

Imports will need to include `MODEL_CONFIGS` and `DEFAULT_MODEL_ID`.

### 3. `src/embedder.py` ‚Äî Multi-model embedder with prefix dispatch

Refactor `HuggingFaceEmbedder` to:
- Accept a `model_id` parameter (resolved to model config)
- Use model config for `model_name`, `dtype`, and `prefixes`
- Dispatch prefix logic based on model config:
  - If `prefixes` is not None (jina): prepend `prefixes["code2code"]["passage"]` for chunks, `prefixes["nl2code"]["query"]` for queries
  - If `prefixes` is None (nomic): no prefix prepending
- Handle dtype based on config (`float16` vs `bfloat16`, with MPS/ROCm fallback)

Update `create_embedder()` to accept `model_id` and resolve config:

```python
def create_embedder(model_id: str | None = None, **kwargs) -> HuggingFaceEmbedder:
    from .config import MODEL_CONFIGS, DEFAULT_MODEL_ID
    model_id = model_id or DEFAULT_MODEL_ID
    config = MODEL_CONFIGS[model_id]
    return HuggingFaceEmbedder(
        model=config["model_name"],
        model_id=model_id,
        **kwargs,
    )
```
